# Модель задачи: дескрипторы и @property
## Цель работы:
Освоить управление доступом к атрибутам и защиту инвариантов доменной
модели.

## Функциональность:
1) использование пользовательских дескрипторов для валидации атрибутов
2) использование property для вычисляемых или защищённых свойств
3) предотвращение некорректных состояний объекта
4) генерация специализированных исключений при нарушении инвариантов

## Задача содержит следующие атрибуты:
- `id` — уникальный идентификатор (неизменяемый)
- `description` — описание задачи
- `priority` — приоритет от 1 до 5
- `status` — статус (pending, in_progress, completed, failed)
- `created_at` — время создания (только для чтения)
- `is_ready` — вычисляемое свойство (готовность к выполнению)
- `age_seconds` — вычисляемое свойство (возраст задачи в секундах)
